שלב 1: גילוי ונחיתה
המטופל מגיע לדף landing-patient.html, קורא על היתרונות והתהליך, ולוחץ על "התחל כעת".
שלב 2: טופס קליטה (ללא התחברות)
מילוי טופס ב-patient-onboarding.html:
  • שם מלא (חובה)
  • טלפון (חובה, 9+ ספרות)
  • עיר מגורים
  • סיבת הפנייה (חובה, מינימום 10 תווים)
  • העדפת סוג טיפול
  • העדפת מגדר מטפל
  • ניסיון קודם בטיפול
שלב 3: סקירת מנהל ושיבוץ
המנהל רואה את הבקשה בדאשבורד, מעריך את הצרכים, ומשבץ מטפל מתאים על פי:
  • התמחות רלוונטית
  • מיקום גיאוגרפי או זמינות לזום
  • העדפת מגדר
  • קיבולת פנויה (פחות מ-10 מפגשים)
שלב 4: שער משפטי וגישה לדאשבורד
לפני גישה לדאשבורד, המטופל חייב לחתום על תנאי השימוש ב-legal-gate.html. לאחר מכן מקבל גישה ל-patient-dashboard.html.
שלב 5: טיפול ומשוב
  • צפייה בפרטי המטפל המשובץ
  • יצירת קשר דרך WhatsApp
  • עד 10 מפגשים עם אותו מטפל
  • דירוג (1-5 כוכבים) לאחר כל מפגש
  • אפשרות למשוב אנונימי

📊 סטטוסים של מטופל

in_treatment בטיפול פעיל מפגשים מתקיימים

  • רקע והכשרה מקצועית

🤖 תהליך סינון AI

Gemini 1.5 Flash מנתח את השאלון המקצועי ומייצר:
  • HIGH - מומלץ לאישור אוטומטי
  • MEDIUM - נדרשת סקירה ידנית
  • LOW - סביר להיות נדחה
  • PENDING - בתהליך ניתוח

🔄 זרימת אישור מטפל

בקשה חדשה (status='new')
    ↓
ניתוח AI (Gemini 1.5 Flash)
    ↓
ציון AI מוצג (HIGH/MEDIUM/LOW)
    ↓
מנהל סוקר:

├── ✅ אישור → status='active' → אימייל קבלה
├── 📞 ראיון → status='pending_interview' → זימון
└── ❌ דחייה → status='rejected' → אימייל דחייה

מטופל חדש (status='waiting_for_match')

├── העדפת מגדר
├── מיקום
    ↓
סינון מטפלים:
├── סטטוס פעיל

בחירה ושיבוץ:
├── עדכון assigned_therapist_id

🔌 8. ממשקי API

כל הפעולות מתבצעות דרך Supabase JavaScript Client. אין שרת Backend נפרד - הלוגיקה רצה בצד הלקוח.

🔐 Authentication

Auth.signUp(email, password, fullName)   // הרשמה
Auth.signIn(email, password)             // התחברות
Auth.signInWithGoogle()                   // Google OAuth
Auth.signOut()                           // יציאה
Auth.getCurrentUser()                     // משתמש נוכחי
Auth.resetPassword(email)                 // איפוס סיסמה

👥 Profiles

Profiles.get(userId)                     // קבלת פרופיל לפי ID
Profiles.getCurrent()                    // פרופיל משתמש נוכחי
Profiles.update(userId, updates)         // עדכון פרופיל
Profiles.uploadAvatar(userId, file)      // העלאת תמונה

👨‍⚕️ Therapists

Therapists.getAll()                      // כל המטפלים הפעילים
Therapists.getById(id)                   // מטפל לפי ID
Therapists.getCurrent()                  // מטפל מחובר
Therapists.register(data)                // יצירת רשומה
Therapists.update(id, updates)           // עדכון
Therapists.getReviews(therapistId)       // ביקורות
Therapists.getAppointments(id, status?)  // פגישות

🧑‍⚕️ Patients

Patients.getCurrent()                    // מטופל מחובר
Patients.register(data)                  // יצירת רשומה
Patients.update(id, updates)             // עדכון
Patients.getAppointments(patientId)      // פגישות
Patients.submitReview(reviewData)        // שליחת ביקורת

📅 Appointments

Appointments.create(appointmentData)     // יצירת פגישה
Appointments.update(id, updates)         // עדכון
Appointments.cancel(id)                  // ביטול
Appointments.complete(id)                // סיום

🎓 Course Progress

CourseProgress.getAll(courseType?)       // כל הסרטונים
CourseProgress.isVideoWatched(videoId)   // האם נצפה
CourseProgress.markVideoWatched(...)     // סימון כנצפה
CourseProgress.getCourseProgress()       // התקדמות כללית

⚖️ שער משפטי (Legal Gate)

חובה: כל משתמש חייב לחתום על תנאי השימוש לפני גישה לדאשבורד מוגן. 
זרימת בדיקה:
1. האם דף ציבורי? → כן: אפשר גישה
2. האם מחובר? → לא: הפניה לindex.html
3. האם חתם על תנאים? → לא: הפניה ל-legal-gate.html
4. האם יש לו תפקיד מתאים? → לא: הפניה לדאשבורד המתאים
5. → כן: אפשר גישה
