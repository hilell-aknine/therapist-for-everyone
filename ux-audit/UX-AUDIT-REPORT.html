טיפול בשגיאות

  • משוב Toast בלבד — אין שגיאות inline ליד השדות. משתמשי מובייל צריכים לגלול למעלה.
  • שגיאת רשת גנרית — "שגיאה בשליחת הטופס. נסה שוב." — ללא פירוט.
  • חסר autocomplete על כל ה-inputs בכל הטפסים.

  • Canvas נמחק בסיבוב מובייל אבל hasSignature נשאר true ← תמונה ריקה נשלחת
  • אין חלופה למשתמשים עם מוגבלות מוטורית (WCAG 2.5.1)

7 ביצועים וארכיטקטורה

HIGH כפילות CSS: ~4,400 שורות

רק 2 קבצי CSS משותפים: theme.css (215 שורות) + accessibility.css (135 שורות).

~80 שורות כפולות לכל דף (tokens, navbar, grain, reveal) × 55 דפים = ~4,400 שורות copy-paste.

חסר: components.css, navbar.css, layout.css

משאבים חוסמי Render

  • Font Awesome נטען סינכרוני ב-6/8 דפים (רק הפורטל טוען async)
  • Supabase SDK נטען סינכרוני ללא preload
  • חסר preconnect ל-Cloudflare CDN

אופטימיזציות חסרות

  • אין loading="lazy" על תמונות
  • אין OG/social meta tags (משפיע על תצוגה מקדימה ב-WhatsApp)
  • Grain overlay עם feTurbulence — עתיר GPU במובייל
